Category 1: My Decision Trail

Why Arduino over Zephyr

I did consider Zephyr RTOS, but for this task it felt like setup-heavy overhead. I wanted faster iteration for sensor parsing and output checks, so Arduino was the practical choice.

I still kept FreeRTOS as a future path if I need tighter scheduling.

Why LwGPS instead of TinyGPS++ (even though TinyGPS++ was suggested)

TinyGPS++ is valid. I picked LwGPS mainly because it felt lighter and easier to reason about in my PlatformIO workflow for this repo. I wanted parsing behavior to stay simple while debugging serial data.

So this was not "TinyGPS++ is bad", it was "LwGPS fits my debugging style better here".

How I reduced confusion while debugging
  • Environment 1: LwGPS-only tests
  • Environment 2: micro-ROS GPS publishing tests

This split helped me isolate failures quickly: parser side vs ROS publish side.
